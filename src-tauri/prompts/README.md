# AI 提示词配置

这个目录包含AI写作助手的提示词配置文件。

## 文件说明

### task_planner.json
**任务规划器提示词**

用于分析用户意图并生成任务执行计划。当用户发送指令时，AI会根据这个提示词判断：
- 是简单问答（answer）
- 还是需要执行文件操作的任务（task）

如果是任务，会生成详细的TODO列表。

### content_generator.json
**内容生成器提示词**

用于生成文件内容。当执行"think"类型的TODO时使用此提示词，例如：
- 创建新文件的内容
- 总结/分析文件
- 生成要插入的内容

## 配置格式

每个配置文件包含三个字段：

```json
{
  "name": "配置名称",
  "description": "配置描述",
  "system_prompt": "实际的提示词内容"
}
```

## 如何修改提示词

1. **直接编辑JSON文件**
   - 修改 `system_prompt` 字段的内容
   - 保存文件后重启应用即生效

2. **注意事项**
   - 保持JSON格式正确
   - `system_prompt` 中的换行使用 `\n`
   - 引号需要转义：`\"`
   - 修改后建议测试功能是否正常

3. **备份**
   - 修改前建议备份原文件
   - 可以创建多个版本进行A/B测试

## 提示词调优建议

### task_planner.json
- 明确任务类型的判断标准
- 提供详细的示例（包含完整的JSON结构）
- 说明每种操作类型的使用场景
- 强调输出格式要求

### content_generator.json
- 强调内容质量和风格要求
- 说明如何处理不同类型的生成任务
- 避免AI输出元信息或解释性文字
- 根据实际使用效果持续优化

## 故障排除

如果提示词文件加载失败：
1. 检查JSON格式是否正确
2. 查看应用日志中的警告信息
3. 应用会自动使用默认提示词作为后备
4. 确保文件编码为UTF-8

## 高级用法

可以在 `src-tauri/src/ai/prompts.rs` 中：
- 添加新的提示词配置文件
- 创建特定场景的提示词变体
- 实现动态提示词选择逻辑
